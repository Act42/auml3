# auml3
# Звіт до лабораторної роботи №3: Система розпізнавання мовлення на Toronto Dataset

## 1. Огляд даних
Для виконання роботи було використано **Toronto Dataset**. 
* **Обсяг даних:** У роботі використано 50% від доступного обсягу датасету (`DATASET_SIZE_PERCENT = 0.5`) для пришвидшення процесу навчання.
* **Розподіл (Splitting):** * Згідно з вимогами, для тестування використано виключно записи з префіксами: `toronto_27, 46, 42, 37, 89, 43, 157, 9, 156, 7, 123, 54, 67, 62, 81, 134, 148, 21, 135, 166, 58`.
    * Решта даних була розділена на тренувальну та валідаційну вибірки у співвідношенні 90/10.
* **Особливості та проблеми:**
    * Дані містять розмовну українську мову з паузами, вигуками ("хм-хм") та специфічною пунктуацією.
    * Необхідність обробки аудіо з частотою дискретизації 16000 Гц для сумісності з моделлю Whisper.

---

## 2. Опис використаних підходів
Для побудови системи ASR (Automatic Speech Recognition) було обрано наступний стек:
* **Базова модель:** `KSE-RESEARCH-Group/whisper-small-dido-yvanchyk-v2` — донавчена версія Whisper-small, що вже має базові знання української мови.
* **Фреймворк:** **PyTorch Lightning**, що дозволив структурувати логіку навчання, логування та використання GPU.
* **Конфігурація навчання:**
    * **Оптимізатор:** AdamW з `learning_rate = 1e-5`.
    * **Епохи:** 2 епохи навчання.
    * **Прецизія:** `16-mixed` (змішана точність) для оптимізації пам'яті GPU.
    * **Процесор:** `WhisperProcessor` для токенізації тексту та вилучення ознак (feature extraction).

## 3. Таблиця метрик
Результати тестування моделі на відкладеній вибірці (test set):

| Метрика | Значення |
| :--- | :--- |
| **WER (Word Error Rate)** | **0.4268** (42.68%) |
| **CER (Character Error Rate)** | **0.2464** (24.64%) |

Що цікаво, при навчанні на 10% модель була точінше щодо символів, але менш точною щодо слів

## 4. Гіпотези та висновки щодо результатів
1. **Аналіз помилок:** Модель демонструє непогані результати на коротких фразах, проте значно втрачає точність на специфічних термінах або словах, які звучать схоже (наприклад, "дирілці" замість "де рєльси").
2. **Пунктуація:** Оригінальні мітки мають складну пунктуацію (тире, трикрапки), яку модель іноді ігнорує або спрощує, що негативно впливає на WER, але менше — на CER.
3. **Обмеження:** Використання лише 50% даних та невелика кількість епох (2) обмежують потенціал моделі. Збільшення вибірки та тривалості навчання ймовірно дозволить знизити WER нижче 30%.
4. **Співвідношення метрик:** CER (24%) значно нижчий за WER (42%), що свідчить про те, що модель правильно розпізнає основу слів, але часто помиляється у закінченнях або частках.
